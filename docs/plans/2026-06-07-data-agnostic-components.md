# Plan: Enforce Data-Agnostic Components

**Date:** 2026-06-07
**Status:** Open
**Scope:** All build phases, scaffold, and supporting docs

## Problem

Components generated by pixel-perfect's build command sometimes contain hardcoded/mock data baked into the component file instead of being isolated in story/sandbox files. This makes components non-reusable and couples rendering logic to fake data.

**Root cause:** The instructions never explicitly *forbid* hardcoded data in component files. The story examples are correct (data comes from args), but no rule says the component can't also contain mock data.

## Fix

Add explicit "DATA-AGNOSTIC COMPONENT RULE" guardrails at every build level and in supporting docs. Every component file must receive ALL display data via props. Story files own ALL mock data.
